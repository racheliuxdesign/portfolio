Filters Makeover

Added greater flexibility for handling more question types. Introduced enterprise-level filter organization for large-scale data.

Role Lead Product Designer
Duration 3 months
Team Designer, 2 Engineers, PM
Hero screenshot placeholder

Overview

The existing filtering system was limited in question types and struggled with large-scale enterprise data. Users needed a more flexible approach to filtering that could handle complex queries while maintaining clarity and speed.

Problem

Enterprise users working with thousands of data points found the existing filter interface inadequate. Common pain points included limited filter types, inability to save or share filter configurations, and poor performance when dealing with complex nested conditions.

Before state — existing filter interface

Design Process

Research involved interviewing power users and analyzing usage patterns. Key insights showed that users needed hierarchical filter grouping, more question types (date ranges, multi-select, custom fields), and the ability to save and reuse filter configurations. Iterative prototyping and usability testing refined the interaction model.

Design exploration — wireframes & prototypes
Final design — new filter interface

Outcome & Impact

The redesigned filter system supported 8+ new question types and introduced enterprise-level organization with saved views and shared filter sets. User satisfaction scores improved significantly, and the time to configure complex filters was reduced by approximately 40%.
