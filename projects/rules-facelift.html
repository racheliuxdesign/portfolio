Rules Facelift

Added new abilities, made rule logic easier to understand, and upgraded the overall visual experience.

Role Senior Product Designer
Duration 2.5 months
Team Designer, 2 Engineers, PM
Hero screenshot placeholder

Overview

The rules engine is a core component of the cybersecurity platform, enabling users to define automated responses to security events. The existing interface was functional but visually outdated and difficult for new users to learn. This project aimed to modernize the experience while adding powerful new capabilities.

Problem

Users struggled to understand complex rule logic, especially when dealing with nested conditions and multiple action chains. The visual presentation didn't clearly communicate the relationship between conditions and actions, leading to configuration errors and reduced confidence in rule behavior.

Before state — original rules interface

Design Process

Conducted heuristic evaluation and user testing of the existing interface. Identified key areas where visual hierarchy could better communicate rule structure. Designed a new visual language for conditions and actions using color coding, indentation, and flow-based layouts. Iterated through multiple rounds of testing with security analysts and administrators.

Design exploration — rule logic visualization
Final design — redesigned rules engine

Outcome & Impact

The redesigned rules interface reduced configuration errors and improved user confidence. New abilities added included conditional branching, template rules, and real-time rule preview. The updated visual design aligned with the modern platform aesthetic and received positive feedback from both new and experienced users.
